## 实测清单

- [ ] 页面能否正常打开（HTTPS）
- [ ] 右上角 `···` 菜单是否出现「发送给朋友」「分享到朋友圈」
- [ ] 分享卡片：标题、描述、缩略图是否正确（OG 生效）
- [ ] 点击页面「分享」按钮：`navigator.share` 是否存在
- [ ] 若不存在：复制链接降级是否成功、引导是否易懂
- [ ] 朋友 B 点开二次链接：工具是否可用、是否愿意再分享（主观）

## 预期结论（Phase 1 假设）

1. 微信内置浏览器大概率不支持 `navigator.share`
2. 降级方案（复制链接 + 右上角引导）应为主路径
3. 被动分享卡片由 `index.html` 静态 OG meta 控制

## 部署命令

```bash
npx vercel login
npx vercel deploy --prod
```

首次部署后，将实际域名写回 `index.html` 的 `og:image` 与 `og:url`，再执行一次部署。
